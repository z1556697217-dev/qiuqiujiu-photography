# 个人写真摄影师 AI 客服助理 — 项目总结

## 一、项目背景

- **身份**：独立摄影师（小红书：秋秋啾爱拍照）
- **业务**：个人写真摄影（主要拍女生）
- **痛点**：微信咨询量大，重复回答报价/档期问题；需要整理客户记录
- **定位**：AI 小助理，不是替代摄影师，而是前台接待 + 后台管理工具人

---

## 二、双号架构

```
┌─ 小号 ───────────────┐      ┌─ 大号（秋秋）──────────┐
│  前台接待 Bot          │      │  后台管理 Bot           │
│                        │      │                        │
│  👤 客户 1v1 或群聊     │      │  💬 秋秋私聊对话         │
│  ✅ 报报价/查档期/发作品 │      │  ✅ @小助理 整理档期    │

│  ✅ 过滤难缠客人        │      │  ✅ @小助理 查客户历史  │
│  ✅ 收集需求→转大号     │      │                        │
│  ❌ 不确认套餐          │      │  小助理人格：            │
│  ❌ 不收定金            │      │  机灵、有底线、会来事    │

└──────────────────────┘      └──────────────────────────┘
              │                          │
              └──────── 必要时 ──────────┘
                   拉三人群转接
              （客户 + 小号 + 大号秋秋）
```

## 三、三阶段生命周期

### 阶段一：小号接客（前台）

| 功能 | 说明 |
|------|------|
| 自动回复 | 报价/档期/作品/闲聊 |
| 客户画像 | 砍价标记、语气标记、需求清晰度 |
| 收集需求 | 风格、时长、精修数量、是否化妆等 |
| 确认意向 | 聊到位后引导转大号 |
| 底线 | ❌ 不确认套餐 ❌ 不收定金 ❌ 不承诺 |

### 阶段二：大号管理（后台 — 秋秋私聊）

| 秋秋说的话 | Bot 理解并执行 |
|-----------|--------------|
| "小助手帮我整理一下档期喔~" | 返回近期档期表 |
| "小红那个改成25号了" | 更新客户预约日期 |
| "这个客人不拍了" | 取消预约，释放档期 |
| "小美那单拍完了" | 标记已完成 |
| "帮我看看下周还有什么空" | 查下周档期 |
| "这个人又在纠缠退定金" | 标记客户 + 通知秋秋 |

### 阶段三：售后归档

拍摄完成后标记客户状态，归档聊天记录，方便后续查询。

## 四、报价体系 — YAML 配置驱动

```yaml
packages:
  基础套餐:
    price: 699
    includes: {时长: 1h, 精修: 9张, 服装: 1套, 场景: 室内}
  标准套餐:
    price: 1299
    includes: {时长: 2h, 精修: 20张, 服装: 2套, 场景: 室内/室外}
  豪华套餐:
    price: 2399
    includes: {时长: 3-4h, 精修: 35张, 服装: 3套, 场景: 室内+室外, 妆造: 含}

addons:
  额外精修: {price: 50/张}
  妆造: {price: 299, condition: 仅基础/标准可选}
  宠物清洁费: {price: 100}

rules:
  - 双人拍摄 → 基础+50%，其他+30%
  - 指定非本店主摄影师 → 上浮20%
```

- 改价格改 YAML，不碰代码
- 客户报需求 → 引擎匹配套餐 + 算加项 → AI 组装话术

## 五、定价/退款场景应对策略（核心）

### 5.1 客户说"家里得癌症/死人了/生病了"

```
第一层 ─ 共情 + 给台阶
  小助理："啊这样……姐姐您别太难过，身体和家人最重要。
           咱们这边可以免费改期一次的，您先忙家里的事，
           想拍了随时找我安排~"
  → 不提退款，引导改期

第二层 ─ 坚持不退
  对方坚持退款 →
  "姐我真的很理解您，但定金不只是我这边收的，
   档期留出来之后秋秋姐的时间就排死了，
   临时退单这边损失也挺大的……
   您看改个时间行吗？任何时候想拍了跟我说一声就行~"

第三层 ─ 冷处理 + 通知秋秋
  连续纠缠 3 次以上 →
  回复拉长间隔，话术变简短：
  "姐这个我真的破不了例，要不我跟秋秋姐说一声，
   让她跟您直接沟通？"
  → 私信秋秋："定金纠纷，已沟通3次，需要您定夺"
```

**核心原则：**
- 不质疑真假，不拆穿，不当判官
- 语气软，立场硬
- 秋秋出面之前 Bot 绝不松口说"我问问能不能退"
- 免费改期一次是 Bot 权限上限

### 5.2 砍价/要赠送

```
客户："便宜点嘛""送几张精修"
→ "这个我没办法做主哦，要问秋秋姐~"
→ 后台标记 #砍价倾向
→ 不主动降价
```

### 5.3 言语攻击/威胁

```
辱骂 → Bot 冷处理（拉长回复间隔）
威胁曝光 → "好的我让主理人跟您沟通"
→ 后台标记 #疑似差评威胁
→ 紧急通知秋秋
```

### 5.4 骚扰/黄色内容

→ Bot 停止回复
→ 🚨 立即通知秋秋

## 六、客户画像系统

每轮对话 Bot 内部标记，不出现在客户面前：

```python
flags = []
# 自动检测
"便宜|贵|打折|优惠|送" → flags: #砍价倾向
"发原片|先看看效果"     → flags: #白嫖试探
"喂|回话|人呢"         → flags: #语气偏冲
"能不能退|退钱|退款"   → flags: #定金纠纷
改方案 ≥2次            → flags: #反复修改
威胁曝光/差评           → flags: #疑似差评威胁
```

转接时附带客户画像：

```
📋 客户转接报告
├─ 套餐：标准套餐 1299
├─ 档期：7月20日 14:00
├─ 砍价次数：3次 ❗
├─ 需求修改：2次
├─ 标签：#砍价倾向 #反复修改
└─ ⚠️ AI建议：价格敏感但态度友好，建议接单后提前确认定金政策
```

## 八、五层校验体系

### Layer 1 — 输入校验

```
目标：过滤无效/有害消息
├── 广告/骚扰/链接 → 静默忽略
├── 识别客户身份 → 新客户/老客户/黑名单
├── 消息类型分类 → 报价/档期/闲聊/攻击性/无法分类
└── @小助理 只在大号私聊触发，群内不主动发言
```

### Layer 2 — 意图置信度

```
目标：AI 判断 + 分级处理
├── 高置信度(>85%) → 自动回复
├── 中等(60-85%) → 自动回复 + 抄送秋秋知悉
├── 低置信度(<60%) → "我转给秋秋姐亲自回复你~"
└── 付费意愿识别 → 高意愿走转接流程
```

### Layer 3 — 业务逻辑校验（硬闸门）

```
目标：AI 不越权，不瞎承诺
⛔ Bot 无权操作：
  ├── 确认套餐/锁定档期
  ├── 收定金/退定金
  ├── 承诺赠送/打折/附加服务
  └── 破例改规则

✅ Bot 可操作：
  ├── 报套餐价格范围
  ├── 告诉客户某日期是否有空（不定死）
  ├── 免费改期一次（超出需秋秋）
  └── 标记客户/打标签

校验规则：
├── 报价金额 → 检查是否在 YAML 价格范围内
├── 档期 → 检查 SQLite 是否冲突（只读不锁）
├── 退款 → 检测到"退"字 → 触发退款话术模板
└── 赠送 → 检测到"送/免/打折" → ⛔ 拒绝
```

### Layer 4 — 回复内容审核

```
目标：AI 生成的内容不出格
├── 不允许自定报价/承诺额外服务
├── 不允许透露秋秋私人信息（电话/地址/身份证号）
├── 不允许替秋秋做决定（"我觉得可以退" → ⛔）
├── 涉及金钱/改规则 → 自动检查是否合规
├── 消极语气/不易读 → 重新生成
└── 所有回复附追踪 ID（可回溯、可撤回）
```

### Layer 5 — 人工兜底

```
目标：关键操作必须有真人参与
├── 预约确认 → 必须秋秋在群里亲自确认才生效
├── 定金 → 秋秋收，Bot 不沾钱
├── 退款 → 秋秋决定，Bot 只守底线不松口
├── 难缠客户 → Bot 顶住三轮，第三轮后通知秋秋
├── 秋秋可随时接管任何对话（拉群/接管/回滚）
├── 所有 AI 回复可追溯（聊天记录 + 时间戳 + 意图置信度）
└── 一键关闭 Bot 自动回复（全人工模式）
```

### 校验全景图

```
消息进来
  │
  ▼
Layer 1 ── 输入校验 ← 过滤广告/骚扰
  │
  ▼
Layer 2 ── 意图置信度 ← AI 打分分级
  │
  ├── 高置信度 ────→ 自动回复
  ├── 中置信度 ────→ 自动 + 通知秋秋
  └── 低置信度 ────→ 转秋秋
          │
          ▼
Layer 3 ── 业务逻辑校验 ← 检查越权操作
  │
  ├── ✅ 合规 → 继续
  └── ⛔ 违规 → 拒绝 + 话术模板
          │
          ▼
Layer 4 ── 回复审核 ← 检查 AI 生成内容
  │
  ├── ✅ 通过 → 发送
  └── ⛔ 违规 → 重生成 / 转模板
          │
          ▼
Layer 5 ── 人工兜底 ← 关键操作摄影师确认
          │
          ▼
        执行/存储
```
